# 安全部署脚本使用说明

`scripts/safe-deploy.sh` 用于服务器上的受保护更新，执行顺序固定为：校验 `docker-compose.yml` 包含 `./storage:/app/storage` 挂载，备份 `storage/` 到带时间戳的目录，拉取最新代码，执行 `docker compose up -d --build`，输出容器状态和最近日志。

默认在仓库内执行：

```bash
cd ~/printk
bash scripts/safe-deploy.sh
```

脚本默认使用当前 Git 分支，默认备份目录为 `storage/backups/storage-YYYYMMDD-HHMMSS/`，备份内容会排除 `storage/backups/` 自身，避免备份目录递归增长。

可配置变量：

```bash
APP_DIR=~/printk BRANCH=master BACKUP_ROOT=storage/backups LOG_TAIL=200 bash scripts/safe-deploy.sh
```

变量含义：

- `APP_DIR`：项目目录，默认取脚本所在仓库根目录。
- `BRANCH`：拉取的分支，默认取当前分支；处于 detached HEAD 时需要显式设置。
- `BACKUP_ROOT`：备份根目录，默认 `storage/backups`。
- `LOG_TAIL`：部署完成后每个服务输出的最近日志行数，默认 `100`。

脚本停止条件：

- `docker-compose.yml` 缺少 `./storage:/app/storage` 挂载。
- `storage/` 备份失败。
- Git 拉取无法快进合并。
- Docker Compose 重建或启动失败。

恢复时从目标备份目录复制文件回 `storage/`，再执行：

```bash
cd ~/printk
sudo docker compose up -d
sudo docker compose ps
```
