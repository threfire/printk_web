# 团队网站扩展设计方案

## 1. 文档目的

本方案用于把当前“物资入库与发票管理系统”扩展为团队统一门户。团队门户承载战队展示、机器人展示、队员风采、功能系统入口和赛季月度规划；发票管理作为门户下的一个业务系统继续保留现有流程。

## 2. 产品目标

团队网站 = 对外展示入口 + 对内工具入口 + 赛季协同入口。

第一阶段目标：

- 成员进入局域网页面后先看到团队首页。
- 首页展示战队介绍、队徽视觉、机器人展示、队员风采和近期动态。
- 首页提供功能系统入口，当前包含发票管理和赛季月度规划。
- 发票管理沿用当前上传、后台审核、入库、出库流程。
- 赛季月度规划支持按月份、按组别查看任务计划。
- 管理员可维护展示内容和规划内容。

## 3. 用户角色

团队成员：浏览团队首页、查看机器人与队员展示、进入发票上传、查看赛季规划。

组长：查看本组月度规划、维护本组计划、更新任务进度。

管理员：维护首页内容、机器人资料、队员资料、功能入口、赛季规划和发票管理后台。

访客：查看公开展示内容。第一阶段支持外网访问，公开内容可直接浏览；涉及管理员后台、组长编辑、成员真实照片和内部规划编辑的页面必须登录。


页面入口路径建议：

| 页面 | 路径 | 访问对象 |
| --- | --- | --- |
| 团队门户首页 | `/` | 全员 |
| 发票管理入口 | `/invoices` | 全员 |
| 发票管理后台 | `/invoices/admin` | 管理员 |
| 赛季月度规划 | `/season-plan` | 全员 |

## 5. 首页设计

首页定位：团队形象页 + 工具入口页。

首屏结构：

- 顶部导航：队徽、战队名称、首页、机器人、队员、赛季规划、发票管理、主题切换。
- 英雄区：战队口号、赛季目标、主视觉机器人或队徽。
- 快捷入口：发票管理、赛季月度规划、成员上传、管理员后台。
- 状态提示：当前服务运行地址、模板下载入口、最近更新时间。

内容区结构：

- 战队介绍：战队名称、成立时间、定位、赛事方向、训练基地。
- 机器人展示：机器人名称、定位、负责组别、核心特性、图片。
- 队员风采：姓名、组别、职责、照片、个人简介。
- 近期动态：训练、比赛、招新、阶段成果。
- 功能中心：以卡片形式进入下级系统。

首页状态：

| 状态 | 页面表现 |
| --- | --- |
| 内容完整 | 展示完整图片区和文字介绍 |
| 图片缺失 | 使用队徽或机械纹理占位 |
| 动态为空 | 显示“暂无近期动态” |
| 功能不可用 | 卡片置灰并显示维护说明 |
| 移动端访问 | 内容单列排列，快捷入口优先展示 |

## 6. 机器人展示设计

机器人展示用于沉淀每赛季机器人资料。

建议字段：

| 字段 | 含义 |
| --- | --- |
| 机器人名称 | 例如英雄、步兵、哨兵、工程 |
| 赛季 | 所属赛季 |
| 负责组别 | 电控组、机械组、算法组等 |
| 机器人定位 | 比赛中的职责 |
| 核心特性 | 结构、电控、算法亮点 |
| 当前状态 | 设计中、装配中、调试中、参赛中、归档 |
| 展示图片 | 首页和详情页使用 |
| 详情说明 | 长文本介绍 |

页面结构：

- 列表页：机器人卡片网格。
- 详情页：大图、基础信息、技术亮点、负责成员。
- 管理页：新增、编辑、隐藏、归档。

## 7. 队员风采设计

队员风采用于展示团队成员和组别构成。

展示策略：队员风采展示真实姓名和真实照片。因为外网访问会扩大可见范围，队员资料管理页需要提供“展示、隐藏”开关。

建议字段：

| 字段 | 含义 |
| --- | --- |
| 姓名 | 成员姓名 |
| 组别 | 电控组、机械组、算法组、运营组等 |
| 职责 | 组内职责 |
| 届别 | 年级或赛季身份 |
| 照片 | 头像或训练照 |
| 简介 | 个人介绍 |
| 展示状态 | 展示、隐藏 |

页面结构：

- 按组别筛选。
- 成员卡片展示。
- 管理员维护资料。

## 10. 赛季月度规划设计

赛季月度规划 = 赛季目标按月份拆解，再按组别落到任务。

核心对象：

- 赛季
- 月份
- 组别
- 任务
- 负责人
- 截止日期
- 当前进度

组别建议：

- 电控组
- 机械组
- 算法组
- 视觉组
- 运营组
- 管理组

任务状态：

| 状态 | 含义 |
| --- | --- |
| 未开始 | 任务已列入计划 |
| 进行中 | 已开始执行 |
| 待验收 | 已完成主要工作，等待确认 |
| 已完成 | 组长或管理员确认完成 |
| 延期 | 超过计划截止日期 |

附件策略：第一阶段赛季任务不支持附件上传。任务资料通过文字说明和备注记录；设计图、表格、会议记录等文件继续放在团队既有文件夹或群文件中。

总览页结构：

- 当前赛季名称
- 当前月份
- 月度目标
- 各组任务完成数
- 延期任务数量
- 组别筛选
- 月份切换

组别视图结构：

- 本组月度目标
- 任务列表
- 负责人
- 截止日期
- 状态标签
- 备注

管理页结构：

- 新建赛季
- 新建月份计划
- 新建组别任务
- 修改任务状态
- 归档历史赛季

## 12. 管理权限设计

第一阶段权限采用简单密码登录。

权限规则：

- 首页浏览不需要登录。
- 发票后台需要管理员密码。
- 团队内容管理需要管理员密码。
- 赛季规划浏览不需要登录。
- 赛季规划编辑需要组长独立密码或管理员密码。
- 组长登录后只能维护本组规划任务。
- 管理员登录后可以维护全部组别规划任务。
- 后续可扩展成员账号。

密码配置：

- 管理员密码继续使用 `ADMIN_PASSWORD`。
- 组长密码新增 `GROUP_LEADER_PASSWORD`。
- 如果需要分系统密码，可新增 `SITE_ADMIN_PASSWORD` 和 `PLAN_ADMIN_PASSWORD`。

## 13. 外网访问设计

外网访问 = 本机服务运行 → 外网入口转发 → 用户通过公网地址访问。

第一阶段建议使用内网穿透或路由器端口映射。内网穿透部署步骤更少，适合先试运行；路由器端口映射需要公网 IP 或运营商支持，适合长期固定访问。

外网访问安全要求：

- 管理后台必须登录。
- 管理员密码和组长密码必须改为强密码。
- 需要配置 HTTPS。
- 上传接口需要限制文件类型和大小。
- 登录接口需要限制连续失败次数。
- 图片和富文本内容需要由管理员审核后展示。
- 数据库和 `storage/` 目录需要定期备份。

访问量边界：

- 当前 Flask + SQLite + 本地文件存储适合团队级低并发访问。
- 几十人日常浏览、少量成员上传表格、组长维护规划可以承受。
- 多人同时上传大文件、外网大量访问图片、频繁刷新后台会造成响应变慢。
- 第一阶段可通过图片压缩、登录限流、上传大小限制和定期备份控制压力。
- 如果后续开放到大量公开访问，再升级为生产 Web 服务部署和更强数据库。

## 14. UI 视觉方向

视觉继续沿用队徽风格：深蓝、红、白、机械感、RoboMaster 赛事感。

主题策略：

- 默认深色主题。
- 保留小太阳按钮切换亮色主题。
- 首页使用更强的战队展示感。
- 下级系统使用更清晰的工具页布局。

视觉元素：

- 队徽作为导航品牌识别。
- 背景使用深蓝网格、红色能量线、金属面板。
- 机器人卡片使用大图和编号标签。
- 功能入口使用高对比按钮。
- 规划页面使用清晰表格和状态标签。

### 首页英雄区

用途：展示战队主形象和核心入口。

包含内容：

用途：进入下级系统。

包含内容：

- 功能名称

- 可用状态显示主色按钮。

用途：查看赛季任务。

包含内容：

交互规则：

### 内容管理表单

用途：维护首页内容。

## 16. 响应式设计

桌面端：

- 首页使用双列首屏。
- 机器人和队员使用卡片网格。
- 赛季规划使用表格和侧边筛选。

- 所有按钮具备可识别文字。

- 深色和亮色主题文字对比度满足阅读需求。
- 状态标签同时使用文字和颜色表达。

## 20. 验收标准

门户验收：

- 局域网成员访问根地址进入团队首页。
- 首页能展示队徽、战队介绍、功能入口。
- 主题切换在首页和下级系统均生效。
- 移动端页面内容可正常阅读和操作。

发票管理验收：

- 成员可从门户进入发票上传。
- 原有模板下载、上传、后台登录、入库、出库流程可用。
- 旧链接能跳转到新入口。

赛季规划验收：

- 成员可按月份查看计划。
- 成员可按组别筛选任务。
- 管理员可新增和修改任务。
- 任务状态在总览中正确统计。

内容管理验收：

- 管理员可维护战队介绍、机器人资料、队员资料、近期动态。
- 近期动态支持富文本编辑。
- 图片上传后能在首页和详情页显示。
- 隐藏内容不会出现在公开页面。
